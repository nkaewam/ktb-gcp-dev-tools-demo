# Agent Guidelines

This document provides instructions and best practices for AI agents working on this project.

## GitHub Operations

> [!IMPORTANT]
> **Always use the GitHub MCP** for any task that requires interaction with the remote repository.

- **Creating Pull Requests**: Use Github MCP to create PR.
- **Pushing Code**: Use git to commit and push changes to the repository.
- **Repository Management**: Use Github MCP to create branches, forking, and managing issues.
- **Code Reviews**: Use Github MCP to review PRs.

## Tech Stack & Development Standards

- **Framework**: Next.js (App Router)
- **Runtime & Package Manager**: Bun
- **Language**: TypeScript
- **Styling**: Tailwind CSS (Utility-first approach)
- **UI Components**: shadcn/ui
  - Check [shadcn/ui](https://ui.shadcn.com/) or the shadcn MCP before creating custom components.
  - Follow existing architectural patterns for custom components.

## Naming Conventions

- **Files**: Use `kebab-case` for all file names (e.g., `user-profile.tsx`).

## Design & Aesthetics

- **Rich Aesthetics**: Interfaces should be modern, vibrant, and premium.
- **Dynamic Design**: Use hover effects, smooth transitions, and micro-animations.
- **No Placeholders**: Use `generate_image` for realistic assets.
